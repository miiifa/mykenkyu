次の実装は、
候補λ選択からJ評価へ。

現在のデモでは、複数のλC候補を試し、再構築スコアが最も良いものを採用しています。次の実装では、説明ページで整理した正則化付き目的関数 J(π, λC) を明示的に計算し、Jが小さい解を採用する形へ寄せます。

このページは、次に index.html を変更するときの実装仕様です。いきなりコードを大きく触る前に、何を計算するかを固定します。

1. 目的関数

J(π, λC)
= α P_hat(π, λC)
+ (1 - α) B_hat(π)
+ β (s(π, λC) - τ)^2
+ γ (λC - λ0)^2
記号意味初期値
α合成駒内部コストと見た目補助評価のバランス0.75
β色寄与率が目標τからズレる罰則0.20
γλCが基準λ0からズレる罰則0.05
τ目標色寄与率0.25
λ0基準色重み0.35
λ範囲λCが極端に逃げるのを防ぐ範囲0.10〜0.80

2. P_hat: 合成駒内部コスト

既存のUIサーチで使っているコストを分解して使います。Y, Cb, Cr の各成分を個別に計算し、λCで色成分を重み付けします。

P_hat_Y(π)  = normalized piece variance for Y
P_hat_Cb(π) = normalized piece variance for Cb
P_hat_Cr(π) = normalized piece variance for Cr

P_hat(π, λC)
= P_hat_Y(π) + λC { P_hat_Cb(π) + P_hat_Cr(π) }
P_hat はスケール差が出やすいので、セル数・画像数・特徴次元で割って正規化してから J に入れる。

3. s: 色寄与率

λCが0に逃げないように、探索コスト中で色成分がどれくらい寄与しているかを測ります。

color = λC { P_hat_Cb(π) + P_hat_Cr(π) }
total = P_hat_Y(π) + color + ε
s = color / total

この s が τ=0.25 に近づくように、β(s-τ)^2 を加えます。

4. B_hat: 見た目補助評価

最初の実装では、B_hat は軽いブロック特徴から始めます。セル内を 2×2 ブロックに分け、合成駒と元セルのブロック平均のズレを見る想定です。

B_hat(π)
= average block mismatch between synthetic pieces and original cells
B_hat は P_hat と同じ「平均との差」だけにしない。局所ブロック、エッジ、隣接連続性など、合成駒内部コストとは違う観点を使う。

5. 探索手順

まずは厳密な同時最適化ではなく、交互最適化で近似します。

initialize π, λC
best = (π, λC, J)

repeat:
  # step A: λC fixed
  run UI search and update π

  # step B: π fixed
  try neighborhood λ candidates
  compute J(π, λC') for each
  choose best λC'

  # step C: accept
  if J improves:
    update best
  else:
    shake π slightly

return best
