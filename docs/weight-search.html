色重み探索は、
UIサーチの目的関数を選ぶ探索。

このページは、デモに入れた「色重みを自動探索」が何をしているかを説明するメモです。結論から言うと、合成駒の内部ばらつきを下げる内側探索と、再構築画像の見た目でλを選ぶ外側探索を分けています。

白黒で探索して最後に色を付けるわけではありません。探索コストの中で、明るさ Y を主役にしつつ、色差 Cb / Cr を λC で弱めに効かせています。

1. 内側の問題: λを固定してUIサーチ

画像数を N、セル数を V とします。画像 i の割り当てを πᵢ として、合成駒 k は各画像から1セルずつ選ばれます。

π*(λ) = argminπ C_piece(π; λ)

ここで C_piece は、同じ合成駒に入るセル同士がどれだけ近いかを見るコストです。UIサーチでは1画像だけを動かし、他画像を固定します。すると普通の割当問題になるので、ハンガリー法または大きい分割ではgreedyで解きます。

2. セル特徴量: YCbCr

Y

明るさ。形、輪郭、濃淡に効くので主成分として扱います。デモでは4×4ブロック平均。

Cb / Cr

色味。色に引っ張られすぎると形が崩れるため、2×2ブロック平均にして λC で補助的に効かせます。

C_k = D_Y(k) + λC D_Cb(k) + λC D_Cr(k)

D_Y, D_Cb, D_Cr は、それぞれ合成駒内での平均との差の二乗和です。次元数の違いで片方だけ強くなりすぎないよう、ベクトル長で割って平均二乗誤差として扱います。

3. 外側の問題: λを変えて再構築スコアで選ぶ

λを変えると、合成駒を作る目的関数そのものが変わります。そのため、同じ C_piece だけでλを比較すると循環します。そこで、外側では固定の再構築スコアを使います。

λ* = argminλ C_recon(π*(λ))

C_recon = E_Y + 0.35 E_Cb + 0.35 E_Cr

つまり、各λでUIサーチを軽く回し、その結果できた合成駒で元画像にどれだけ戻せるかを見て、最も良いλを採用します。

4. デモで試しているλ候補

λC意味起こりやすいこと
0.00グレースケール探索明暗や形を重視。色は探索上は無視される。
0.15〜0.30弱めの色補助形を大きく壊さず、少し色を合わせる。
0.45〜0.60中〜強めの色補助色の印象は近づくが、形がやや崩れる可能性がある。
0.80かなり色重視カラフルな画像では効く可能性があるが、色に引っ張られやすい。

5. 次の拡張: ブロックモデル再配置

今の重み探索は、合成駒の作り方を改善する方向です。一方で、最終的な見た目をさらに良くするには、作った合成駒をどの位置に置くかも重要です。

次の本命は、合成駒生成は UIサーチ + YCbCr で行い、その後にブロックモデルで再配置することです。これは「合成駒として良いか」と「絵として戻せるか」を分ける方針です。
合成駒生成: UIサーチ + YCbCr + λ探索
再配置補正: ブロックモデルで、元画像の局所的なムラやグラデーションに近づける
