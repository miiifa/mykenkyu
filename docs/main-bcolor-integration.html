メインデモへ、
B_colorを安全に統合する。

PR #19で実画像セルから Lab ΔE ベースの B_color を計算できることを確認した。次はメインデモの docs/index.html にこの評価を統合する。ただし index.html は大きく壊れやすいため、まず差し込み関数と表示項目を固定する。

この仕様ページは、次の実装PRで index.html を触るためのチェックリストです。目的は、全面置換ではなく関数単位で安全に差し込むことです。

1. 追加する特徴量

現在のセル特徴は Y, Cb, Cr を持っている。ここに Lab 平均を追加する。

cell.feature = {
  y: [...],
  cb: [...],
  cr: [...],
  lab: { l, a, b }
}

Labはセル全画素の平均ではなく、各画素をLabへ変換してから平均する。

2. 追加する関数

関数役割差し込み先
srgbToLinearsRGBを線形RGBへ変換する。rgbToYCbCr の近く
rgbToXyz線形RGBからD65 XYZへ変換する。色変換群
xyzToLabXYZからCIELABへ変換する。色変換群
rgbToLabRGBからLabへ変換する。splitImageIntoCells 内で使用
deltaE76Lab間の色差を計算する。B_color 計算で使用
meanLab複数Labの平均を返す。B_color 計算で使用
bColorCostmapSet全体のB_color平均を計算する。runWeightSearch の評価値

3. B_color の定義

B_color(π)
= mean_k mean_i { ΔE76(Lab_i,k, meanLab_k)^2 / 10000 }

k は合成駒スロット、i は画像番号。小さいほど、同じ合成駒に入るセル同士が色知覚上近い。

4. J評価への入れ方

B(π) = η1 B_color(π) + η2 B_edge(π)

J(π, λC)
= α P(π, λC)
+ (1 - α) B(π)
+ β(s(π, λC)-τ)^2
+ γ(λC-λ0)^2

最初は B_edge を既存の簡易値または0として扱い、B_color を先に統合する。SSIM風構造差やGMSD風エッジ差は次段階にする。

B_color を入れても、UIサーチ本体の割当コスト P を急に置き換えない。まずは λ候補の採用基準と表示値に J / B_color を入れる。

5. 表示する値

表示名意味
J正則化付き総合目的関数。採用判定に使う。
P合成駒内部コスト。既存のUIサーチ目的。
B_colorLab ΔEベースの色知覚コスト。
s色寄与率。
(s-τ)^2色寄与率ペナルティ。
(λ-λ0)^2λ正則化ペナルティ。

6. 実装順

  1. index.html に RGB→Lab 関数群を追加する。
  2. splitImageIntoCells で cell.feature.lab を保存する。
  3. bColorCost(mapSet) を追加する。
  4. objectiveJ(mapSet, lambda, pieceCost) を追加する。
  5. runWeightSearch の row に J, B_color, s, penalties を追加する。
  6. 採用基準を score から J へ切り替える。
  7. weightResults テーブルに J / B_color を表示する。
