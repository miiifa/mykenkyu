作業ログ

  • PR #1〜#3 初期デモと合成駒生成

    まずは静的な研究紹介ページと、GitHub Pagesで動くデモの土台を作成。3枚画像を分割し、各画像から1セルずつ選んでRGB平均の合成駒を作る構造を実装した。

  • PR #4 UIサーチ風の改善ループ

    合成駒の内部ばらつきをコストとして、1画像だけを動かす割当問題に分解。ハンガリー法を使って、固定された他画像に対して選択画像の割り当てを改善する流れを作った。

  • PR #5〜#8 研究ページ・探索管理・可視化

    研究紹介ページを追加し、UIサーチの試行回数、採用、棄却、改善率などを見えるようにした。分割数に応じた探索回数や、ランダム初期解とハンガリー初期解の比較も整理した。

  • PR #10 リプレイ機能

    UIサーチの採用点をスナップショットとして保存し、初期解からベスト解までを自動再生できるようにした。後で、探索状態を壊さないpreview専用リプレイへ修正する必要があることも分かった。

  • PR #11 YCbCrと色重み探索

    探索コストをグレースケールのみから、Y, Cb, Cr に分解した特徴へ拡張。Yを主成分、Cb/Crを補助成分として扱い、λCを候補探索で選ぶプロトタイプを実装した。UIサーチ本体と局所脱出も分離した。

  • PR #12 色重み探索の説明ページ

    λC探索の説明ページを追加。内側でUIサーチ、外側で再構築スコア評価という二段階の考え方を整理した。

  • PR #13 定式化のレビューと方針修正

    λCを単純に目的関数へ入れると0へ逃げる問題、再構築コストが合成駒コストと同じになりうる問題、再配置変数をUIサーチ本体に混ぜすぎる問題を確認。正則化付きの交互最適化として整理し直した。

  • PR #14〜#15 J評価デモと実装仕様

    J(π, λC) の各項、初期係数、色寄与率、UI表示値、変更対象関数を仕様ページとして整理し、ミニデモで J を計算できるようにした。

  • PR #16 見え方評価の参考文献整理

    CIEDE2000、色知覚ベースの類似度、SSIM、GMSD、ΔE_NS、LPIPSを、B_color / B_structure / B_edge / B_boundary のどこに使うか整理した。

  • 今回 B_color に Lab ΔE を導入

    J評価ミニデモの B に、sRGB → XYZ → CIELAB 変換と ΔE76 ベースの B_color を追加した。次はこれをメインデモの重み探索へ統合する。

途中で分かった注意点

気づき対応
λCを自由に最小化すると、色を無視する λC=0 に逃げる。λCの範囲制約、色寄与率の正則化、基準値λ0からのズレ罰則を入れる。
合成駒コストと再構築コストが同じ特徴だと、二観点最適化にならない。見た目側はブロック特徴、エッジ、隣接連続性など別特徴を見る。
再配置変数をUIサーチ本体に混ぜると、問題設定が分かりにくくなる。合成駒生成とブロックモデル再配置を二段構成として説明する。
リプレイが探索状態を直接変更すると、探索本体が壊れやすい。previewDataで表示専用にする。
デモ本体を触る前に、目的関数仕様を固定した方がよい。J評価仕様ページを作り、次の実装対象を明確にする。
YCbCrの色距離だけだと、人間の色知覚とはズレる可能性がある。まずB_colorにLab ΔEを入れ、将来的にCIEDE2000へ発展させる。
